# 配送遅延予測アプリ開発課題

## 依頼内容

あなたは通販会社「Hoshi Market」のデータチームに所属しています。

最近、お客様から「荷物が届くのが遅い」という問い合わせが増えました。配送担当者は、発送前に遅延の可能性を確認したいと考えています。

そこで、過去の配送データから配送状態を予測するAIアプリを作ることになりました。

完成するアプリは、配送情報を入力すると、次の3つの状態を表示します。

| 結果 | 意味 |
|---|---|
| `通常` | 通常どおり配送できる可能性が高い |
| `遅延注意` | 遅延する可能性があるため確認が必要 |
| `遅延危険` | 大きく遅れる可能性が高い |


## 配布ファイル

| ファイル | 説明 |
|---|---|
| `delivery_dirty.csv` | 30,000行・17列の汚れた配送データ |
| `01_clean_data.py` | データをきれいにするプログラム（自分で作成） |
| `02_train_model.py` | AIモデルを学習・保存するプログラム（自分で作成） |
| `03_delivery_app.py` | 予測を行うCLIアプリ（自分で作成） |
| `generate_delivery_data.py` | データ作成用。通常は実行しない |
| `requirements.txt` | 必要なライブラリの一覧 |

## 必須処理

1. `delivery_clean.csv`を読み込む
2. 使用するカテゴリ列を数値へ変換する
3. 自分で選んだ特徴量から`X`を作成し、`y`に配送状態を入れる
4. `train_test_split()`で学習用とテスト用に分ける
5. `stratify=y`を指定する
6. `DecisionTreeClassifier`を学習する
7. 正解率を表示する
8. モデル・特徴量・使用したコード表をまとめて保存する


## 課題：CLIアプリを作る

`03_delivery_app.py`を作成してください。

アプリは`delivery_model.pkl`を読み込み、ユーザーが入力した配送情報から結果を予測します。

### 入力する項目

- 自分が選んだすべての特徴量

CLIアプリの入力項目は、`FEATURES`と同じにしてください。カテゴリ列を使用した場合は、入力された文字をモデル用の数値へ変換してください。

### 必須機能

1. `joblib.load()`でモデルを読み込む
2. メニューを表示する
3. 数値以外が入力された場合に、もう一度入力させる
4. 天気は`晴れ`、`雨`、`雪`以外なら入力し直させる
5. 入力値をpandasのDataFrameにする
6. `model.predict()`で予測する
7. 結果に応じたメッセージを表示する
8. 終了メニューを作る

### 表示例

```text
=== 配送遅延予測アプリ ===
1: 予測する
2: 終了
選択してください: 1

距離 (km): 650
荷物重量 (kg): 18
天気 (晴れ・雨・雪): 雪
交通量 (0〜100): 90
発送準備時間 (h): 20
過去遅延回数: 5

予測結果: 遅延危険
大きく遅れる可能性があります。配送方法を確認してください。
```

## 実行する順番

```bash
python3 03_delivery_app.py
```
