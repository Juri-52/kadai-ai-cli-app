# 配送遅延予測アプリ開発課題

## 依頼内容

あなたは通販会社「Hoshi Market」のデータチームに所属しています。

最近、お客様から「荷物が届くのが遅い」という問い合わせが増えました。配送担当者は、発送前に遅延の可能性を確認したいと考えています。

そこで、過去の配送データから配送状態を予測するAIアプリを作ることになりました。

完成するアプリは、配送情報を入力すると、次の3つの状態を表示します。

| 結果 | 意味 |
|---|---|
| `通常` | 通常どおり配送できる可能性が高い |
| `遅延注意` | 遅延する可能性があるため確認が必要 |
| `遅延危険` | 大きく遅れる可能性が高い |

## 制作時間

**3時間**

| 列名 | 内容 | 問題の例 |
|---|---|---|
| `注文ID` | 注文番号 | 同じIDの重複行がある |

| `距離` | 配送距離（km） | `120km`、空欄、`不明`、負の値、異常値がある |
| `荷物重量` | 荷物の重さ（kg） | `5kg`、空欄、異常値がある |
| `天気` | 配送時の天気 | `晴れ`、`雨`、`雪`、英語、余分な空白がある |
| `交通量` | 道路の混雑度（0〜100） | 空欄、`不明`、負の値、100より大きい値がある |
| `発送準備時間` | 発送準備に必要な時間（h） | `3h`、空欄、異常値がある |

| `当日配送件数` | その日に担当する配送件数 | `件`、空欄、異常値がある |
| `ドライバー経験年数` | 担当ドライバーの経験年数 | `年`、空欄、異常値がある |
| `再配達回数` | 受取人不在などによる再配達回数 | `回`、空欄、異常値がある |
| `時期` | 通常期または繁忙期 | `通常期`、`繁忙期`、英語、余分な空白がある |
| `配送先種別` | 住宅または会社 | 日本語、英語、余分な空白がある |
| `配達希望時間帯` | 午前・午後・夜間 | 日本語、英語、余分な空白がある |
| `配送状態` | 正解ラベル | 余分な半角・全角空白がある |

## 課題1：データをきれいにする

`01_clean_data.py`を作成してください。

### 必須処理

1. `delivery_dirty.csv`をpandasで読み込む
2. 列名の空白を削除する
3. `注文ID`の重複を削除する
4. `地域`、`配送会社`、`天気`の表記を統一する
5. 数値の列から`km`、`kg`、`h`、`円`、`件`、`年`、`回`、カンマ、空白を取り除く
6. `不明`と空欄を欠損値に変換する
7. 数値に変換できない値を欠損値に変換する
8. 異常値を欠損値に変換する
9. 欠損値を各列の中央値で埋める
10. `配送状態`の空白を削除する
11. `通常`、`遅延注意`、`遅延危険`以外の行を削除する
12. `delivery_clean.csv`として保存する

### 異常値のルール

| 列名 | 正常な範囲 |
|---|---|
| `距離` | 0より大きく3,000以下 |
| `荷物重量` | 0より大きく100以下 |
| `交通量` | 0〜100 |
| `発送準備時間` | 0より大きく48以下 |
| `過去遅延回数` | 0〜30 |
| `注文金額` | 0より大きく1,000,000以下 |
| `当日配送件数` | 0より大きく500以下 |
| `ドライバー経験年数` | 0〜40 |
| `再配達回数` | 0〜20 |

### 確認する出力

次のような情報を表示してください。

```text
清掃前: (30000, 17)
清掃後: (..., 17)

配送状態
通常        ...
遅延注意    ...
遅延危険    ...
```

## 課題2：AIモデルを学習・保存する

`02_train_model.py`を作成してください。

使用するモデルは`DecisionTreeClassifier`です。

### 特徴量を自分で選ぶ

この課題では、モデルに使用する特徴量を**自分で選択**してください。

最初は、数値の列を6個以上選んでください。例：

```python
FEATURES = [
    "距離",
    "荷物重量",
    "交通量",
    "発送準備時間",
    "過去遅延回数",
    "当日配送件数",
    "再配達回数",
]
```

その後、別の列を追加・削除して正解率を比べてください。

| 列の種類 | 使用するために必要な処理 |
|---|---|
| 数値列 | 清掃後、そのまま使用できる |
| `天気` | `晴れ`、`雨`、`雪`を数値に変換する |
| `時期` | `通常期`、`繁忙期`を数値に変換する |
| `配送先種別` | `住宅`、`会社`を数値に変換する |
| `配達希望時間帯` | `午前`、`午後`、`夜間`を数値に変換する |
| `地域`、`配送会社` | 発展課題：自分で数値化の方法を考える |

### カテゴリ列のコード化の例

天気などの文字列は数値に変換してください。

```python
weather_map = {
    "晴れ": 0,
    "雨": 1,
    "雪": 2,
}
```

### 必須処理

1. `delivery_clean.csv`を読み込む
2. 使用するカテゴリ列を数値へ変換する
3. 自分で選んだ特徴量から`X`を作成し、`y`に配送状態を入れる
4. `train_test_split()`で学習用とテスト用に分ける
5. `stratify=y`を指定する
6. `DecisionTreeClassifier`を学習する
7. 正解率を表示する
8. モデル・特徴量・使用したコード表をまとめて保存する

保存形式の例：

```python
bundle = {
    "model": model,
    "features": FEATURES,
    "maps": {"weather_map": weather_map},
}
joblib.dump(bundle, "delivery_model.pkl")
```

## 課題3：CLIアプリを作る

`03_delivery_app.py`を作成してください。

アプリは`delivery_model.pkl`を読み込み、ユーザーが入力した配送情報から結果を予測します。

### 入力する項目

- 自分が選んだすべての特徴量

CLIアプリの入力項目は、`FEATURES`と同じにしてください。カテゴリ列を使用した場合は、入力された文字をモデル用の数値へ変換してください。

### 必須機能

1. `joblib.load()`でモデルを読み込む
2. メニューを表示する
3. 数値以外が入力された場合に、もう一度入力させる
4. 天気は`晴れ`、`雨`、`雪`以外なら入力し直させる
5. 入力値をpandasのDataFrameにする
6. `model.predict()`で予測する
7. 結果に応じたメッセージを表示する
8. 終了メニューを作る

### 表示例

```text
=== 配送遅延予測アプリ ===
1: 予測する
2: 終了
選択してください: 1

距離 (km): 650
荷物重量 (kg): 18
天気 (晴れ・雨・雪): 雪
交通量 (0〜100): 90
発送準備時間 (h): 20
過去遅延回数: 5

予測結果: 遅延危険
大きく遅れる可能性があります。配送方法を確認してください。
```
